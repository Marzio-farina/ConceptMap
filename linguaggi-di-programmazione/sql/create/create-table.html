CREATE TABLE

Il comando CREATE TABLE in SQL è utilizzato per creare una nuova tabella in un database. Quando si crea una tabella, si specifica il nome della tabella e i nomi delle colonne, insieme ai loro tipi di dati e vincoli, se necessario.

Sintassi di Base

CREATE TABLE nome_tabella (
    nome_colonna1 tipo_dato vincolo_opzionale,
    nome_colonna2 tipo_dato vincolo_opzionale,
    ...
);

Esempi di CREATE TABLE

  1. Creare una Tabella Semplice

    Supponiamo di voler creare una tabella per memorizzare informazioni sugli studenti.

CREATE TABLE studenti (
    id INT PRIMARY KEY,
    nome VARCHAR(50),
    cognome VARCHAR(50),
    data_di_nascita DATE,
    email VARCHAR(100) UNIQUE
);

In questo esempio:

  • id è una colonna di tipo intero (INT) che funge da chiave primaria (PRIMARY KEY), il che significa che il suo valore deve essere univoco per ogni record nella tabella.
  • nome e cognome sono colonne di tipo stringa (VARCHAR), ciascuna con una lunghezza massima di 50 caratteri.
  • data_di_nascita è una colonna di tipo data (DATE).
  • email è una colonna di tipo stringa con una lunghezza massima di 100 caratteri ed è unica (UNIQUE), il che significa che nessun altro record nella tabella può avere lo stesso valore per email.

  1. Creare una Tabella con Vincoli

    Un esempio più complesso potrebbe includere vincoli come FOREIGN KEY, DEFAULT, NOT NULL, ecc.

CREATE TABLE ordini (
    ordine_id INT PRIMARY KEY,
    cliente_id INT,
    data_ordine DATE DEFAULT CURRENT_DATE,
    importo DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (cliente_id) REFERENCES clienti(cliente_id)
);

In questo esempio:

  • ordine_id è la chiave primaria.
  • cliente_id è un intero che rappresenta l'ID del cliente. È una chiave esterna (FOREIGN KEY) che fa riferimento alla colonna cliente_id della tabella clienti.
  • data_ordine è di tipo DATE e, se non specificato, assume per impostazione predefinita (DEFAULT) la data corrente (CURRENT_DATE).
  • importo è di tipo decimale con precisione totale di 10 cifre e 2 cifre decimali. Il vincolo NOT NULL assicura che questo campo non possa essere lasciato vuoto.

  1. Creare una Tabella con Auto Increment

    Se si desidera che una colonna aumenti automaticamente il suo valore per ogni nuovo record, è possibile utilizzare AUTO_INCREMENT (in MySQL) o SERIAL (in PostgreSQL).

-- Esempio in MySQL
CREATE TABLE prodotti (
prodotto_id INT AUTO_INCREMENT PRIMARY KEY,
nome VARCHAR(100),
prezzo DECIMAL(10, 2)
);

-- Esempio in PostgreSQL
CREATE TABLE prodotti (
prodotto_id SERIAL PRIMARY KEY,
nome VARCHAR(100),
prezzo NUMERIC(10, 2)
);

In questi esempi:

  • prodotto_id è una colonna di tipo intero che si incrementa automaticamente con ogni nuovo inserimento.
  • nome è una colonna stringa.
  • prezzo è una colonna decimale/numerica per memorizzare i prezzi dei prodotti.

Questi esempi mostrano come il comando CREATE TABLE può essere utilizzato per creare tabelle con diverse caratteristiche e vincoli, a seconda delle esigenze del database.
