CREATE VIEW

Il comando CREATE VIEW in SQL viene utilizzato per creare una vista. Una vista è una tabella virtuale che si basa sul risultato di una query. Le viste possono essere utilizzate per semplificare query complesse, nascondere la complessità dei dati o fornire una visualizzazione specifica dei dati senza modificare le tabelle sottostanti.

Sintassi di Base

CREATE VIEW nome_vista AS
SELECT colonna1, colonna2, ...
FROM tabella
WHERE condizione;

Esempi di CREATE VIEW

  1. Creare una Vista Semplice
    Supponiamo di voler creare una vista che mostri solo il nome e l'email degli studenti.
CREATE VIEW vista_studenti AS
SELECT nome, email
FROM studenti;

In questo esempio:

  • vista_studenti è il nome della vista.
  • La vista mostra solo le colonne nome e email dalla tabella studenti.
  1. Creare una Vista con Filtraggio
    Se si desidera filtrare i dati nella vista, ad esempio, mostrando solo gli studenti nati dopo il 2000.
CREATE VIEW vista_studenti_nati_dopo_2000 AS
SELECT nome, cognome, data_di_nascita
FROM studenti
WHERE data_di_nascita > '2000-01-01';

In questo esempio:

  • vista_studenti_nati_dopo_2000 è il nome della vista.
  • La vista mostra solo gli studenti nati dopo l'1 gennaio 2000.
  1. Creare una Vista con Join

Se si desidera combinare dati provenienti da più tabelle, ad esempio, unire gli ordini con le informazioni sui clienti.

CREATE VIEW vista_ordini_clienti AS
SELECT o.ordine_id, o.data_ordine, c.nome AS cliente_nome
FROM ordini o
JOIN clienti c ON o.cliente_id = c.cliente_id;

In questo esempio:

  • vista_ordini_clienti è il nome della vista.
  • La vista combina le tabelle ordini e clienti mostrando l'ID dell'ordine, la data dell'ordine e il nome del cliente.
  1. Creare una Vista con Aggregazione

Se si desidera creare una vista che mostri i totali delle vendite per ogni cliente.

CREATE VIEW vista_totali_vendite AS
SELECT c.cliente_id, c.nome, SUM(o.importo) AS totale_vendite
FROM ordini o
JOIN clienti c ON o.cliente_id = c.cliente_id
GROUP BY c.cliente_id, c.nome;

In questo esempio:

  • vista_totali_vendite è il nome della vista.
  • La vista mostra l'ID del cliente, il nome del cliente e il totale delle vendite per ciascun cliente, aggregando i dati dalla tabella ordini.

Note Aggiuntive

  • Le viste possono essere utilizzate per semplificare le query, migliorare la sicurezza limitando l'accesso ai dati e facilitare la manutenzione.
  • Le viste non memorizzano i dati fisicamente ma si basano sulla query sottostante per generare il risultato ogni volta che vengono interrogate.
  • È possibile aggiornare una vista, ma le modifiche devono essere compatibili con la struttura della vista e le tabelle sottostanti. Alcuni database hanno restrizioni sui tipi di aggiornamenti che possono essere effettuati tramite una vista.

Questi esempi mostrano come utilizzare il comando CREATE VIEW per creare viste che forniscono una rappresentazione specifica dei dati in un database.
