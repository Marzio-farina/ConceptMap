AndroidJNI

Nel namespace UnityEngine.Android, la classe AndroidJNI fornisce un'interfaccia per interagire con il sistema Java Native Interface (JNI) di Android direttamente da C#. Questa classe è fondamentale per operazioni avanzate che richiedono l'accesso diretto alle API Java di Android, oltre a quelle fornite tramite AndroidJavaObject e AndroidJavaClass.

Dettagli sulla Classe AndroidJNI

  • UnityEngine.Android/AndroidJNI
    • Descrizione: Fornisce metodi statici per accedere e interagire con il sistema JNI di Android. Consente di chiamare metodi Java, accedere a campi e gestire oggetti Java direttamente utilizzando JNI.
    • Metodi Principali:
      • AttachCurrentThread(): Attacca il thread corrente al JVM (Java Virtual Machine) per consentire l'accesso alle operazioni JNI.
      • DetachCurrentThread(): Distacca il thread corrente dalla JVM. Utilizzato per liberare risorse quando il thread non deve più interagire con la JVM.
      • FindClass(string name): Trova e restituisce un riferimento alla classe Java specificata dal nome della classe.
      • GetStaticMethodID(IntPtr classPtr, string methodName, string sig): Ottiene un ID di metodo statico dalla classe Java. Utilizzato per chiamare metodi statici di una classe Java.
      • CallStaticVoidMethod(IntPtr classPtr, IntPtr methodID, params jvalue[] args): Chiama un metodo statico Java e restituisce void. jvalue è un array di argomenti per il metodo.
      • CallStaticObjectMethod(IntPtr classPtr, IntPtr methodID, params jvalue[] args): Chiama un metodo statico Java e restituisce un oggetto.
      • GetFieldID(IntPtr classPtr, string fieldName, string sig): Ottiene un ID di campo dalla classe Java. Utilizzato per accedere ai campi di una classe Java.
      • GetStaticIntField(IntPtr classPtr, IntPtr fieldID): Ottiene il valore di un campo statico di tipo int.
      • SetStaticIntField(IntPtr classPtr, IntPtr fieldID, int value): Imposta il valore di un campo statico di tipo int.
      • NewStringUTF(string bytes): Crea un nuovo oggetto String Java a partire da una stringa C#.
      • GetStringUTFChars(IntPtr str, out IntPtr isCopy): Ottiene i caratteri UTF-8 di una stringa Java come array di byte.
    • Costruttori e Proprietà:
      • Non ci sono costruttori pubblici per AndroidJNI, poiché si tratta di una classe statica.
      • La classe fornisce accesso a una serie di metodi statici, ma non espone proprietà o campi pubblici.

Esempio di Utilizzo

Ecco un esempio di come utilizzare AndroidJNI per chiamare un metodo statico di una classe Java e ottenere un valore da un campo statico:

  1. Definire la Classe Java:
// Java code
package com.example;

public class MyJavaClass {
    public static int staticField = 10;

    public static void staticMethod() {
        System.out.println("Metodo statico chiamato!");
    }
}
  1. Utilizzare AndroidJNI in C#:
using UnityEngine;

public class AndroidJNIExample : MonoBehaviour
{
    void Start()
    {
        // Attacca il thread corrente al JVM
        AndroidJNI.AttachCurrentThread();

        // Trova la classe Java
        IntPtr classPtr = AndroidJNI.FindClass("com/example/MyJavaClass");

        // Ottieni l'ID del metodo statico
        IntPtr methodID = AndroidJNI.GetStaticMethodID(classPtr, "staticMethod", "()V");

        // Chiama il metodo statico
        AndroidJNI.CallStaticVoidMethod(classPtr, methodID);

        // Ottieni l'ID del campo statico
        IntPtr fieldID = AndroidJNI.GetStaticFieldID(classPtr, "staticField", "I");

        // Ottieni il valore del campo statico
        int fieldValue = AndroidJNI.GetStaticIntField(classPtr, fieldID);
        Debug.Log("Valore del campo statico: " + fieldValue);

        // Distacca il thread corrente dalla JVM
        AndroidJNI.DetachCurrentThread();
    }
}

Riassunto

La classe AndroidJNI nel namespace UnityEngine.Android fornisce un'interfaccia diretta per interagire con le API JNI di Android. Permette di chiamare metodi Java, accedere a campi, e gestire stringhe e altri tipi di dati utilizzando il sistema JNI. Questo è particolarmente utile per operazioni avanzate e per l'integrazione con codice Java nativo che richiede un controllo più fine rispetto a quanto offerto da AndroidJavaObject e AndroidJavaClass.
