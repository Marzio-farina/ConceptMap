InputSystem pre 1.8.0

Il namespace UnityEngine.InputSystem fa parte dell' Input System di Unity, che è un sistema più avanzato e flessibile per gestire gli input rispetto al sistema di input classico di Unity. Questo sistema permette di gestire in modo più efficace input da tastiera, mouse, controller, touch screen, e dispositivi personalizzati, ed è particolarmente utile per lo sviluppo di giochi che devono funzionare su più piattaforme.

Principali Funzionalità di UnityEngine.InputSystem

  1. Gestione Avanzata degli Input:

    • Consente di mappare facilmente più dispositivi di input a specifiche azioni di gioco, come il movimento, l'attacco, o l'interazione.
    • Supporta la rimappatura dinamica degli input, permettendo ai giocatori di personalizzare i controlli.

  2. Supporto Multidispositivo:

    • Gestisce input da tastiere, mouse, controller di gioco (come Xbox, PlayStation, ecc.), joystick, dispositivi touch, e altro.
    • Permette di rilevare automaticamente nuovi dispositivi di input collegati durante il runtime.

  3. Azioni di Input:

    • Utilizza il concetto di "Input Actions" che possono essere configurate e gestite per controllare il comportamento di gioco.
    • Le azioni possono essere personalizzate per rispondere a singoli input, combinazioni di tasti, o sequenze di input.

  4. Eventi di Input:

    • Fornisce un sistema basato su eventi che può essere utilizzato per rispondere agli input in modo asincrono.

  5. Schemi di Controllo Personalizzati:

    • Supporta la creazione di schemi di controllo personalizzati che possono essere utilizzati per mappare input specifici per vari dispositivi.

  6. Supporto per l'Input Contextuale:

    • Permette di gestire gli input in contesti specifici, come diversi stati del gioco o modalità di controllo (ad esempio, modalità menu vs. modalità gioco).

Principali Classi e Componenti

  • InputAction: Rappresenta un'azione di input che può essere attivata da uno o più dispositivi di input.
  • InputActionMap: Una raccolta di InputAction che può essere abilitata o disabilitata come un gruppo.
  • InputDevice: La classe base per tutti i dispositivi di input, come tastiere, controller, ecc.
  • Keyboard: Una sottoclasse di InputDevice che rappresenta una tastiera.
  • Mouse: Una sottoclasse di InputDevice che rappresenta un mouse.
  • Gamepad: Una sottoclasse di InputDevice che rappresenta un controller di gioco.
  • Touchscreen: Rappresenta un dispositivo touchscreen.
  • InputActionAsset: Un asset che contiene un set di InputActionMap configurati, spesso creato e configurato tramite l'editor di Unity.
  • PlayerInput: Un componente che facilita l'uso delle azioni di input all'interno di un'architettura basata su Unity.
  • InputProcessor: Classe base per processori di input che modificano o filtrano i valori degli input.
  • InputControl: Classe base per tutti i controlli di input, gestisce la lettura dei valori degli input.
  • InputControlLayout: Gestisce la configurazione dei layout dei controlli di input.
  • InputState: Rappresenta lo stato corrente di tutti i controlli.
  • DeviceConfiguration: Contiene informazioni sulla configurazione di un dispositivo di input.
  • InputActionReference: Riferimento a un'azione di input, utilizzato per riferirsi e attivare azioni.
  • InputBinding: Rappresenta una mappatura tra un'azione di input e un controllo specifico.
  • InputEvent: Base per eventi di input, utilizzata per gestire e rispondere agli input.
  • InputDeviceMatcher: Utilizzato per cercare e identificare dispositivi di input specifici.
  • InputProcessor TValue: Classe generica per i processori di input che modificano valori di input di tipo specifico.
  • InputControlList TControl: Una lista di controlli di input.

Esempio di Configurazione e Utilizzo

Di seguito un esempio di come configurare e usare il nuovo Input System in Unity:

using UnityEngine;
using UnityEngine.InputSystem;

public class PlayerController : MonoBehaviour
{
    public InputAction moveAction;

    private void OnEnable()
    {
        // Abilita l'azione di input
        moveAction.Enable();
    }

    private void OnDisable()
    {
        // Disabilita l'azione di input
        moveAction.Disable();
    }

    void Update()
    {
        // Ottiene il valore dell'azione di input
        Vector2 move = moveAction.ReadValue<Vector2>();

        // Usa il valore dell'input per muovere il giocatore
        transform.Translate(move * Time.deltaTime);
    }
}

Come Usare il Nuovo Input System

  1. Aggiungere il Pacchetto: Il nuovo Input System deve essere aggiunto tramite il Package Manager di Unity.
  2. Configurazione: Configura gli schemi di input e le azioni utilizzando l'editor di Unity.
  3. Scripting: Usa le classi e i metodi del namespace UnityEngine.InputSystem per gestire gli input nel codice.

Considerazioni

  • Compatibilità: Il nuovo Input System è più avanzato ma richiede un po' di configurazione rispetto al sistema di input classico di Unity.
  • Flessibilità: Offre molta flessibilità e potenza, rendendolo adatto per progetti complessi e multi-piattaforma.
  • Documentazione e Risorse: Unity fornisce documentazione dettagliata e tutorial per aiutare gli sviluppatori a migrare al nuovo sistema e sfruttarne appieno le potenzialità.

Riferimenti Aggiuntivi

Per una gestione dettagliata dei controlli dei vari dispositivi di input, puoi fare riferimento al namespace correlato: Controls.

Il namespace UnityEngine.InputSystem è essenziale per chiunque voglia creare giochi con controlli avanzati, gestire input da vari dispositivi, o offrire un'esperienza di gioco più personalizzata.
