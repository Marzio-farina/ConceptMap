Eccezioni Python

In Python, le eccezioni sono eventi che interrompono il normale flusso di esecuzione di un programma. Si verificano quando qualcosa di inaspettato accade, come cercare di dividere per zero o accedere a un elemento di una lista che non esiste.

Perché gestirle?

Gestire le eccezioni è fondamentale per:

  • Prevenire arresti anomali del programma: Invece di far crashare il programma, puoi catturare l'eccezione e eseguire un'azione alternativa.
  • Fornire messaggi d'errore significativi: Puoi personalizzare i messaggi d'errore per aiutare l'utente a capire cosa è andato storto.
  • Separare il codice di gestione degli errori dal codice principale: Questo rende il codice più leggibile e manutenibile.

Come gestire le eccezioni?

Utilizziamo i blocchi try e except per gestire le eccezioni:

try:
    # Codice che potrebbe generare un'eccezione
    risultato = 10 / 0
except ZeroDivisionError:
    print("Non puoi dividere per zero!")
  • try: Il codice all'interno di questo blocco viene eseguito normalmente.
  • except: Se si verifica un'eccezione all'interno del blocco try, il programma salta al blocco except corrispondente.

Tipi di eccezioni:

  • Eccezioni built-in: Python fornisce molte eccezioni predefinite.
  • Eccezioni personalizzate: Puoi creare le tue eccezioni personalizzate ereditando dalla classe base Exception.

Elenco delle eccezioni predefinite:

  • SyntaxError Se c'è un errore nella sintassi del codice.
  • IndentationError: Se l'indentazione del codice è errata.
  • NameError: Se viene utilizzato un nome di variabile o funzione non definito.
  • TypeError: Se viene eseguita un'operazione su un tipo di dato non valido.
  • ValueError: Se una funzione riceve un argomento con il tipo corretto ma un valore inaspettato.
  • ZeroDivisionError: Se si tenta di dividere per zero.
  • IndexError: Se si tenta di accedere a un elemento di una sequenza (come una lista o una stringa) utilizzando un indice non valido.
  • KeyError: Se si tenta di accedere a una chiave non esistente in un dizionario.
  • FileNotFoundError: Se si tenta di aprire un file che non esiste.

Esempio con eccezione personalizzata:

class ErroreEtà(Exception):
    pass

età = int(input("Inserisci la tua età: "))
if età < 0:
    raise ErroreEtà("L'età non può essere negativa")

Altri blocchi:

  • else: Viene eseguito solo se non si verifica alcuna eccezione nel blocco try.
  • finally: Viene sempre eseguito, indipendentemente dal fatto che si sia verificata un'eccezione o meno. È utile per rilasciare risorse, come chiudere file o connessioni al database.

Esempio completo:

def dividi(x, y):
    try:
        risultato = x / y
    except ZeroDivisionError:
        print("Non puoi dividere per zero!")
    else:
        print("Il risultato è:", risultato)
    finally:
        print("Operazione completata")

dividi(10, 0)

Buone pratiche:

  • Cattura solo le eccezioni che ti aspetti: Non catturare tutte le eccezioni in modo generico.
  • Fornisci messaggi d'errore chiari e informativi: Aiuta l'utente a capire cosa è andato storto.
  • Utilizza le eccezioni in modo appropriato: Non utilizzare le eccezioni per il controllo del flusso normale del programma.
  • Crea eccezioni personalizzate quando necessario: Per definire errori specifici del tuo dominio.

Conclusione:

La gestione delle eccezioni è una parte fondamentale della programmazione Python. Ti permette di scrivere codice più robusto, affidabile e facile da mantenere.
