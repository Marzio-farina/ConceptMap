Metodi Animator

Il componente Animator in Unity è centrale per il controllo delle animazioni dei GameObject. Offre una serie di metodi e proprietà che ti permettono di gestire e controllare le animazioni tramite script. Ecco un'analisi dettagliata dei metodi e delle proprietà principali dell'Animator:

Metodi dell'Animator

  1. SetBool Unity (string name, bool value)

    • Imposta un parametro di tipo bool nell'Animator Controller. Utilizzato per attivare o disattivare transizioni tra stati di animazione.
    • Esempio:
    animator.SetBool("IsRunning", true);
  1. SetFloat Unity (string name, float value)
    • Imposta un parametro di tipo float nell'Animator Controller. Utilizzato per gestire animazioni basate su valori numerici, come la velocità.
    • Esempio:
	animator.SetFloat("Speed", 5.0f);

  1. SetInteger Unity (string name, int value)

    • Imposta un parametro di tipo int nell'Animator Controller. Utile per gestire animazioni che dipendono da valori interi, come livelli di stato o modalità.
    • Esempio
	animator.SetInteger("WeaponType", 1);
  1. SetTrigger Unity (string name)
  • Imposta un parametro di tipo trigger nell'Animator Controller. I trigger sono usati per attivare transizioni tra stati di animazione in modo che avvengano solo una volta.
  • Esempio:
	animator.SetTrigger("Attack");

  1. ResetTrigger Unity (string name)

    • Resetta un parametro di tipo trigger nell'Animator Controller. Annulla l'effetto del trigger specificato.
    • Esempio:
	animator.ResetTrigger("Attack");

  1. Play Unity (string stateName, int layer = -1, float normalizedTime = float.NegativeInfinity)

    • Riproduce un'animazione specifica all'interno di un layer e, facoltativamente, a un tempo normalizzato specifico.
    • Esempio:
	animator.Play("JumpAnimation");

  1. CrossFade Unity (string stateName, float transitionDuration, int layer = -1, float normalizedTime = float.NegativeInfinity)

    • Esegue una transizione graduale verso un altro stato di animazione.
    • Esempio:
	animator.CrossFade("RunAnimation", 0.1f);

  1. PlayInFixedTime Unity (string stateName, int layer = -1, float fixedTime = 0)

    • Riproduce un'animazione in un momento specifico del tempo fisso.
    • Esempio:
	animator.PlayInFixedTime("WalkAnimation", 0, 0.5f);

  1. GetBool Unity (string name)

    • Restituisce il valore corrente di un parametro di tipo bool dell'Animator Controller.
    • Esempio
	bool isRunning = animator.GetBool("IsRunning");

  1. GetFloat Unity (string name)

    • Restituisce il valore corrente di un parametro di tipo float dell'Animator Controller.
    • Esempio
	float speed = animator.GetFloat("Speed");

  1. GetInteger Unity (string name)

    • Restituisce il valore corrente di un parametro di tipo int dell'Animator Controller.
    • Esempio:
	int weaponType = animator.GetInteger("WeaponType");

  1. GetTrigger Unity (string name)

    • Verifica se un parametro di tipo trigger è attivo.
    • **Esempio**
	bool isTriggerSet = animator.GetTrigger("Attack");

  1. Update Unity (float deltaTime)

    • Forza l'aggiornamento del sistema di animazione. Normalmente, l'animazione viene aggiornata automaticamente ad ogni frame, ma questo metodo può essere utilizzato se necessario.
    • Esempio
	animator.Update(Time.deltaTime);
  1. StringToHash Unity
	int hash = Animator.StringToHash(string name);
  1. SetLayerWeight (int layerIndex, float weight)
	animator.SetLayerWeight(0, 1.0f); // Imposta il peso del primo layer a 100%

  1. GetLayerWeight Unity

    • Ottiene il peso (weight) per uno specifico layer. I pesi dei layer controllano l'influenza di ogni layer sull'animazione finale.
    • Esempio:
	float layerWeight = animator.GetLayerWeight(0); // Ottiene il peso del primo layer
  1. GetCurrentAnimatorStateInfo Unity
  • Ottiene informazioni sullo stato corrente dell'Animator per un layer specificato.
  • Esempio:
	AnimatorStateInfo stateInfo = animator.GetCurrentAnimatorStateInfo(0); // Ottiene lo stato del primo layer

  1. GetNextAnimatorStateInfo Unity

    • Ottiene informazioni sullo stato successivo dell'Animator per un layer specificato, che può essere utile per animazioni che si stanno preparando a passare allo stato successivo.
    • Esempio:
	AnimatorStateInfo nextStateInfo = animator.GetNextAnimatorStateInfo(0); // Ottiene il prossimo stato del primo layer

  1. GetBoneTransform Unity

    • Ottiene il Transform per un determinato osso nell'Avatar. Questo è utile per accedere ai singoli ossa per scopi di animazione o manipolazione.
    • Esempio:
	Transform headTransform = animator.GetBoneTransform(HumanBodyBones.Head);
  1. StartPlayback Unity
	animator.StartPlayback(); // Avvia la modalità playback
  1. StopPlayback Unity
	animator.StopPlayback(); // Ferma la modalità playback
  1. StartRecording Unity
	// Inizia la registrazione per la durata specificata
	animator.StartRecording((int) durataRegistrazione);
  1. StopRecording Unity
	animator.StopRecording(); // Ferma la registrazione

Esempio di Utilizzo dei Metodi dell'Animator

Immagina di avere un personaggio che può camminare, correre e saltare. Ecco come potresti controllare le animazioni tramite script:

using UnityEngine;

public class PlayerController : MonoBehaviour
{
    public Animator animator;

    void Update()
    {
        float move = Input.GetAxis("Vertical");
        animator.SetFloat("Speed", Mathf.Abs(move));

        if (Input.GetKeyDown(KeyCode.Space))
        {
            animator.SetTrigger("Jump");
        }
    }
}

In questo script:

  • SetFloat("Speed", Mathf.Abs(move)) controlla la velocità dell'animazione in base all'input verticale dell'utente.
  • SetTrigger("Jump") attiva l'animazione di salto quando viene premuto il tasto spazio.

Considerazioni Avanzate

  • Animator Override Controller: Permette di sostituire le animazioni in un Animator Controller esistente con nuove animazioni senza modificare l'Animator Controller originale.
  • Animator Layers: Utilizza i layer per sovrapporre animazioni e gestire animazioni simultanee (come attacchi durante la corsa).
  • Avatar Masks: Usa le maschere dell'Avatar per applicare animazioni solo a specifiche parti del corpo.
  • Playable API: Utilizza l'API Playable per controllare e manipolare le animazioni in modo più avanzato e programmabile.

Conclusione

Il componente Animator di Unity è un potente strumento per la gestione delle animazioni, con una vasta gamma di metodi e proprietà che ti permettono di controllare e personalizzare il comportamento delle animazioni. Utilizzando i metodi dell'Animator, puoi creare animazioni dinamiche e reattive che arricchiscono l'esperienza di gioco.
