ls

Il comando ls in Git Bash, così come in altri ambienti Unix-like, viene utilizzato per elencare i file e le directory presenti nella directory corrente o in una directory specificata. È uno dei comandi più fondamentali e utilizzati per esplorare il contenuto del filesystem.

Uso di Base di ls

Quando esegui ls senza opzioni, il terminale mostra un elenco dei nomi di file e directory nella directory corrente:

	$ ls
	file1.txt  file2.txt  directory1  directory2

Opzioni Comuni di ls

Il comando ls supporta una varietà di opzioni che modificano il modo in cui i file e le directory vengono elencati. Ecco alcune delle opzioni più utili:

  • -l (long format): Mostra i dettagli dei file in un formato lungo, inclusi i permessi, il numero di collegamenti, il proprietario, il gruppo, la dimensione del file e la data di modifica:
	$ ls -l
	-rw-r--r--  1 username  staff   1024 Aug 12 12:34 file1.txt
	drwxr-xr-x  5 username  staff    160 Aug 12 12:34 directory1
  • -a (all files): Elenca tutti i file, inclusi quelli nascosti (che iniziano con un punto .):
	$ ls -a
	.  ..  .hiddenfile  file1.txt  directory1
  • -h (human-readable): Usato in combinazione con -l per mostrare le dimensioni dei file in un formato leggibile (KB, MB, GB):
	$ ls -lh
	-rw-r--r--  1 username  staff   1.0K Aug 12 12:34 file1.txt
	drwxr-xr-x  5 username  staff   160B Aug 12 12:34 directory1
  • -R (recursive): Elenca i contenuti di tutte le directory, ricorsivamente:
	$ ls -R
	.:
	directory1  file1.txt
	
	./directory1:
	subdir1  subfile1.txt
  • -t (time): Ordina i file per data di modifica, dal più recente al meno recente:
	$ ls -lt
  • -r (reverse): Inverte l'ordine di visualizzazione (utile con -t per vedere i file più vecchi per primi):
	$ ls -ltr
  • --color=auto: In Git Bash, i file e le directory sono spesso mostrati in diversi colori per facilitare la distinzione. Questa opzione (attiva per default) abilita il colore automatico.

Combinazione di Opzioni

Puoi combinare le opzioni per personalizzare l'output secondo le tue esigenze. Ad esempio:

	$ ls -lha

Questo comando mostra un elenco dettagliato (-l), leggibile (-h), e include file nascosti (-a).

Usare ls con Percorsi Specifici

Puoi usare ls per elencare i file in una directory diversa specificando il percorso:

	$ ls /path/to/directory

Differenze con Windows CMD

In un terminale di Windows (CMD), l'equivalente del comando ls è dir. Tuttavia, ls è il comando standard nei terminali Unix-like, inclusi Git Bash, macOS Terminal, e Linux shell.

Considerazioni Finali

Il comando ls è uno strumento fondamentale per esplorare e gestire il filesystem in un ambiente Unix-like come Git Bash. Imparare a utilizzare le varie opzioni di ls può migliorare notevolmente la tua efficienza nel lavoro con la riga di comando.
