mv source destination

Il comando mv in Git Bash, così come in altri ambienti Unix-like, viene utilizzato per spostare o rinominare file e directory. È un comando essenziale per la gestione dei file, permettendo di riorganizzare il filesystem senza creare copie ridondanti.

Uso di Base di mv

Il comando mv può essere utilizzato sia per spostare che per rinominare file o directory:

  • Spostare un file o una directory:
$ mv [source] [destination]

Esempio:

$ mv file1.txt /percorso/destinazione/

Questo comando sposta `file1.txt` nella directory `/percorso/destinazione/`.
  • Rinominare un file o una directory:
$ mv [source] [new-name]

Esempio:

$ mv file1.txt file2.txt

Questo comando rinomina `file1.txt` in `file2.txt`.

Spostare o Rinominare Più File

Puoi utilizzare mv per spostare più file contemporaneamente in una directory specificata:

$ mv file1.txt file2.txt /percorso/destinazione/

Questo comando sposta entrambi file1.txt e file2.txt nella directory /percorso/destinazione/.

Opzioni Comuni di mv

Il comando mv supporta diverse opzioni per modificare il suo comportamento:

  • -i (interactive): Chiede conferma prima di sovrascrivere un file di destinazione esistente. Utile per evitare sovrascritture accidentali:
$ mv -i file1.txt /percorso/destinazione/

Verrà chiesto "mv: overwrite '/percorso/destinazione/file1.txt'?" e potrai rispondere `y` (sì) o `n` (no).
  • -f (force): Forza la sovrascrittura di file di destinazione esistenti senza chiedere conferma. Questo è utile in script automatizzati dove la conferma interattiva non è desiderata:
$ mv -f file1.txt /percorso/destinazione/
  • -n (no-clobber): Non sovrascrive mai i file di destinazione esistenti. Se il file esiste già, mv non farà nulla:
$ mv -n file1.txt /percorso/destinazione/
  • -v (verbose): Mostra i file mentre vengono spostati o rinominati, fornendo un feedback visivo dell'operazione:
$ mv -v file1.txt /percorso/destinazione/
mv: 'file1.txt' -> '/percorso/destinazione/file1.txt'

Spostare Directory

mv può anche essere utilizzato per spostare intere directory. Il comando seguente sposta una directory e tutto il suo contenuto (file e sottodirectory):

$ mv /percorso/origine/ /percorso/destinazione/

Esempi di Uso

  1. Spostare un singolo file:
$ mv documento.txt /home/user/Documenti/
  1. Rinominare un file:
$ mv vecchio-nome.txt nuovo-nome.txt
  1. Spostare più file in una directory:
$ mv file1.txt file2.txt /home/user/Documenti/
  1. Spostare una directory e tutto il suo contenuto:
$ mv /home/user/Immagini/ /home/user/Backup/
  1. Spostare un file e chiedere conferma prima di sovrascrivere un file esistente:
$ mv -i file1.txt /home/user/Documenti/
  1. Rinominare una directory:
$ mv vecchia-cartella nuova-cartella

Differenze con Windows CMD

In Windows CMD, il comando equivalente per spostare o rinominare file è move:

C:\> move file1.txt D:\destinazione\

C:\> move vecchio-nome.txt nuovo-nome.txt

Considerazioni Finali

Il comando mv è molto versatile e viene utilizzato frequentemente sia per spostare file e directory, sia per rinominarli. Può essere utilizzato in combinazione con altre opzioni per personalizzare il comportamento del comando in base alle esigenze specifiche.
