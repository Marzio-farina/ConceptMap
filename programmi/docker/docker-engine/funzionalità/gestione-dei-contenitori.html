Comandi di Base per la Gestione dei Contenitori

  • Elencare i contenitori:
    • docker ps: Elenca i contenitori in esecuzione.
    • docker ps -a: Elenca tutti i contenitori, inclusi quelli fermati.
  • Avviare un contenitore:
    • docker start <nome_contenitore>: Avvia un contenitore precedentemente fermato.
  • Fermare un contenitore:
    • docker stop <nome_contenitore>: Arresta gentilmente un contenitore.
  • Rimuovere un contenitore:
    • docker rm <nome_contenitore>: Rimuove un contenitore fermato.
  • Entrare in un contenitore:
    • docker exec -it <nome_contenitore> bash: Entra in un contenitore in esecuzione per eseguire comandi.

Esempi Pratici

# Avvia un contenitore da un'immagine e lo nomina "my_app"
docker run -d --name my_app my_image:latest

# Elenca tutti i contenitori
docker ps -a

# Entra nel contenitore "my_app"
docker exec -it my_app bash

# Ferma il contenitore "my_app"
docker stop my_app

# Rimuove il contenitore "my_app"
docker rm my_app

Gestione Avanzata dei Contenitori

  • Visualizzazione dei log:
    • docker logs <nome_contenitore>: Visualizza i log di un contenitore.
  • Ispezione dei dettagli:
    • docker inspect <nome_contenitore>: Visualizza dettagli completi su un contenitore.
  • Gestione delle reti:
    • docker network: Gestisce le reti create per i contenitori.
  • Gestione dei volumi:
    • docker volume: Gestisce i volumi per persistere i dati dei contenitori.
